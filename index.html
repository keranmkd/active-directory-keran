Vue d’ensemble

Objectif du projet

Centraliser l’administration

Déployer un contrôleur de domaine capable de gérer les utilisateurs, les groupes, les ordinateurs et les ressources d’une entreprise depuis une console unique.

Sécuriser les ressources

Mettre en place des partages réseau maîtrisés avec permissions de partage et NTFS, afin de contrôler précisément l’accès aux dossiers métiers.

Automatiser avec les GPO

Appliquer des stratégies ciblées pour personnaliser l’environnement utilisateur et restreindre certains accès selon le service.

Architecture

Environnement technique

Composants déployés

  • Oracle VirtualBox pour l’environnement de virtualisation
  • Windows Server 2025 pour le serveur principal
  • Active Directory Domain Services (AD DS)
  • Service DNS associé au domaine
  • Postes Windows 11 joints au domaine
  • Partages SMB avec sécurité NTFS

Organisation logique

  • Domaine : keran.local
  • OU : Comptabilité, IT, Direction, Marketing, Ressources humaines, Ordinateurs
  • Groupes : groupes métiers dédiés
  • Ordinateurs : ex. PC-COMPTA1, PC-COMPTA2
  • Dossiers : commun, comptabilité, fond d’écran GPO

Installation

GPO et validation

Déploiement du fond d’écran, blocage ciblé du panneau de configuration et tests finaux.

Validation finale

Tests réussis

  • Connectivité réseau validée entre les machines
  • Jonction des postes clients au domaine keran.local
  • Connexion avec des comptes Active Directory
  • Accès autorisé au dossier commun
  • Accès refusé au dossier comptabilité pour les profils non autorisés
  • Déploiement du fond d’écran via GPO
  • Blocage du panneau de configuration sur les utilisateurs ciblés

Problèmes traités

  • Paramétrage IP / DNS et validation de la communication
  • Permissions de partage et permissions NTFS
  • Accès au chemin réseau du fond d’écran GPO
  • Mise à jour des stratégies via gpupdate
  • Vérification des droits différents selon les services
